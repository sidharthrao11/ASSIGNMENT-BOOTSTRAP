Q.3 What are the default Bootstrap text settings?
Ans :- Bootstrap provides default text settings to ensure consistent and visually appealing typography across your website.
These settings include font size, line height, font weight, and font family.

1.Font Size:Bootstrap sets the base font size to 16 pixels (1rem) for the body element.
It uses relative units like rem and em for font sizes to ensure scalability and responsiveness.

2.Font Family:The default font family for Bootstrap is the system font stack, which varies depending on the user's operating system and browser.
Bootstrap encourages the use of a simple and readable font stack, such as "Helvetica Neue", Helvetica, Arial, sans-serif.

3.Font Weight:Bootstrap applies a default font weight of 400 (normal) for most text elements.
It provides utility classes like .font-weight-bold to apply a font weight of 700 (bold) when needed.

4.Line Height:Bootstrap sets the default line height to 1.5 for improved readability and spacing between lines of text.
This line height ensures that text is well-spaced and easy to read across different screen sizes.

5.Text Color:Bootstrap sets the default text color to #212529 for body text.
It provides utility classes for text color manipulation, such as .text-primary, .text-danger, etc., to apply different colors to text elements.

6.Text Alignment:Bootstrap aligns text to the left (text-left), center (text-center), right (text-right), or justify (text-justify) by default.
You can use these utility classes to align text as needed within your layout.

Q.4 What do you know about the Bootstrap Grid System?
Ans:- The Bootstrap Grid System is a responsive, mobile-first grid system that allows developers to create complex layouts with ease.
It is based on a 12-column grid layout, which provides a flexible and powerful way to design responsive websites. Here's what you need to
know about the Bootstrap Grid System:-

1. **12-column Grid:**
- The grid system is divided into 12 equal columns, allowing for easy division and alignment of content.
- Columns are placed within rows, and the total sum of column widths in a row should not exceed 12.


2. **Responsive Layout:**
- Bootstrap's grid system is mobile-first, meaning it's designed to work on mobile devices first and then scale up to larger screens.
- Columns automatically adjust their width and layout based on the screen size, ensuring a responsive design.


3. **Containers:**
- The grid system should be placed within containers (`container` or `container-fluid` classes) to provide padding and center the content on the page.
- Containers create a fixed-width or full-width wrapper around the grid layout.


4. **Rows:**
- Rows (`row` class) are horizontal wrappers for columns.
- Columns should always be placed within rows to ensure proper alignment and spacing.


5. **Column Classes:**
- Bootstrap provides a set of predefined classes to create columns of various widths.
- Classes such as `col-`, `col-sm-`, `col-md-`, `col-lg-`, and `col-xl-` are used to define column widths at different breakpoints (extra small, small, medium, large, and extra large).
- For example, `col-md-6` creates a column that spans 6 out of 12 columns on medium-sized screens and above.


6. **Offset and Nesting:**
- Bootstrap allows for offsetting columns (`offset-`) to create additional spacing between - Columns can also be nested within other columns to create more complex layouts.


7. **Alignment and Ordering:**
- Bootstrap provides utility classes for aligning columns vertically (`align-items-*`, `align-self-*`, `align-content-*`) and horizontally (`justify-content-*`).
- Columns can also be reordered using `order-*` classes.


8. **Utility Classes:**
- Bootstrap offers additional utility classes for quickly adjusting column widths, hiding/showing columns, and controlling visibility at different screen sizes.


By leveraging the Bootstrap Grid System, developers can create responsive and flexible layouts that adapt to various screen sizes and devices, resulting in a better user experience.
Q.5 What is the difference between Bootstrap 4 and Bootstrap 5?
Ans:- Bootstrap 5, released in November 2020, introduced several changes and improvements compared to Bootstrap 4.
Here's a summary of the key differences between Bootstrap 4 and Bootstrap 5:-
1.jQuery Dependency: -
-> Bootstrap 4 relied on jQuery for some of its JavaScript functionality, such as dropdowns, modals, and tooltips.
->Bootstrap 5 has removed the jQuery dependency, making it lighter and more modern. JavaScript plugins are now written in Vanilla JavaScript or rely on third-party libraries like Popper.js.

2.Customization Options:-
->Bootstrap 5 offers enhanced customization options compared to Bootstrap 4.
->The new version provides utilities for customizing colors, spacing, typography, and borders, allowing developers to easily tailor the design to their needs without needing to write custom CSS.

3.Improved Grid System:-
->Bootstrap 5 retains the 12-column grid system from Bootstrap 4 but introduces new features and improvements.
->The grid system has been enhanced with vertical spacing utilities (gy-* classes) and column gap support, providing more flexibility in creating layouts.

4.Responsive Font Sizes:-
->Bootstrap 5 introduces a new responsive font sizing feature using rem units, allowing font sizes to scale with the viewport width.
This feature helps improve accessibility and readability on different devices and screen sizes.

5.Dropped Internet Explorer Support:-
Bootstrap 5 no longer supports Internet Explorer (IE) 10 and 11, focusing on modern browsers and reducing compatibility issues.
This change enables the use of modern CSS features and improves overall performance.

6. New Components and Utilities:-
->Bootstrap 5 introduces several new components and utilities, including off-canvas navigation, accordion component, list group customization, and improved form validation styles.
->It also provides updated and redesigned versions of existing components for a more modern look and feel.

7.Documentation and Examples:-
->Bootstrap 5 documentation has been revamped and improved, providing clearer explanations, updated examples, and better organization.
->The documentation also includes more guidance on best practices and accessibility considerations.


Overall, Bootstrap 5 builds upon the foundation of Bootstrap 4 while addressing user feedback, improving performance, and modernizing the framework for the web development community.
It offers enhanced customization options, improved documentation, and a more lightweight and flexible approach to building responsive websites.
Q.6 What is a Button Group, and what is the class for a basic Button Group?
Ans:- A Button Group is a UI component used in graphical user interfaces to group related buttons together.
This grouping is typically done to visually associate the buttons and indicate that they belong to the same set
or perform similar actions. Button groups are commonly used in toolbars, forms, or any interface where multiple related actions need to be presented.

The class for a basic Button Group depends on the programming framework or library you're using. For example, in web development with HTML and CSS, you might use the div element
to create a button group and style it accordingly. In frameworks like Bootstrap, there might be specific classes for creating button groups, such as btn-group.

In graphical user interface (GUI) libraries like Swing for Java or Tkinter for Python, there are classes specifically designed for creating button groups. In Swing, for instance,
the ButtonGroup class is used to manage a set of buttons so that only one button in the group can be selected at a time. In Tkinter, you might use a combination of Radiobutton widgets
to achieve similar functionality within a button group.

Q.8 In Bootstrap 4, what is flexbox?
